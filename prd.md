# PRD: LET'S DAPA

## 1. 제품 개요

- 제품명: LET'S DAPA
- 제품 형태: 서비스형 원페이지 웹/앱 랜딩
- 제품 성격: `링크 허브`가 아니라 `콘텐츠 탐색형 서비스 페이지`
- 핵심 목표: 방위사업청 및 정부과천청사 관련 주요 서비스를 로고, 자동 요약, 해시태그, CTA 중심의 카드 섹션으로 큐레이션해 탐색성과 사용성을 높인다.
- 배포 목표: Vercel

## 2. 배경 및 문제 정의

현재 `dapa.md`에 정리된 서비스 후보들은 모두 유용하지만, 단순 링크 나열만으로는 다음 문제가 남는다.

- 서비스처럼 느껴지지 않는다.
- 어떤 링크가 어떤 용도인지 빠르게 파악하기 어렵다.
- 사용자가 페이지 안에서 탐색하는 재미가 부족하다.
- 반복 방문을 유도할 브랜드 인상이 약하다.

따라서 LET'S DAPA는 단순 링크 모음이 아니라 다음 특징을 가져야 한다.

- 서비스별 로고 또는 대표 이미지 제공
- 한눈에 이해되는 자동 요약 제공
- 핵심 키워드를 해시태그로 시각화
- 카드 섹션을 스크롤하며 탐색하는 구조
- 첫 화면에서 분위기를 결정하는 히어로 섹션 제공

## 3. 제품 목표

- 주요 외부 서비스들을 하나의 브랜드 랜딩으로 재구성한다.
- 각 서비스의 목적을 `요약 + 태그 + CTA`로 즉시 이해하게 만든다.
- 사용자가 스크롤하면서 탐색하는 경험 자체를 서비스처럼 느끼게 한다.
- 모바일에서 앱처럼 저장하고 다시 여는 습관을 만든다.

## 4. 타겟 사용자

- 방위사업청 및 유관 조직 구성원
- 정부과천청사 근무자 또는 방문자
- 방위사업 법령, 국회 정보, 청사 생활 정보를 자주 확인하는 사용자
- 모바일에서 빠르게 서비스형 랜딩을 사용하는 사용자

## 5. 제품 포지셔닝

LET'S DAPA는 다음의 중간 지점을 지향한다.

- 공공/업무 유틸리티 허브
- 큐레이션 매거진형 카드 랜딩
- 모바일 친화형 브랜드 페이지

즉, `공공 정보 접근성`과 `에디토리얼 디자인 감도`를 같이 가져가는 제품이다.

## 6. 레퍼런스 해석

### 6.1 TAPTAP 레퍼런스 해석

참고 사이트:

- https://taptap.inpix.com/Magazine
- https://taptap.inpix.com/Archive
- https://taptap.inpix.com/About

관찰 포인트:

- 흰 배경과 넓은 여백을 기반으로 한 에디토리얼 무드
- 강한 제목과 절제된 레이아웃
- 해시태그를 통한 핵심 키워드 구조화
- 좌우 분할 또는 목록형 카드 구조
- 콘텐츠를 스크롤하며 넘겨보는 매거진형 리듬

LET'S DAPA에 적용할 요소:

- 서비스 카드마다 `제목 / 요약 / 해시태그 / CTA / 로고`
- 카드 리스트를 스크롤하며 탐색하는 구조
- 아카이브 느낌의 정렬과 탐색 흐름
- 정보성 서비스도 콘텐츠처럼 보이게 하는 편집 레이아웃

### 6.2 GDWEB 레퍼런스 해석

참고 사이트:

- https://www.gdweb.co.kr/main/index.asp

관찰 포인트:

- 첫 화면에서 시선을 잡는 히어로 역할이 분명함
- 여러 콘텐츠가 있어도 상단 진입부에서 전체 톤을 먼저 제시함
- 큐레이션, 추천, 선정작 같은 `발견형 UX`에 강점이 있음

LET'S DAPA에 적용할 요소:

- 히어로 섹션에서 서비스 정체성을 강하게 전달
- `오늘 가장 먼저 볼 서비스` 또는 `지금 필요한 DAPA 링크` 같은 큐레이션 메시지 사용
- 스크롤 시작 전 사용자가 바로 행동할 수 있는 1차 CTA 제공

## 7. 핵심 기능 정의

## 7.1 히어로 섹션

목적:

- 페이지 첫인상 형성
- 브랜드명과 성격 전달
- 주요 액션 진입점 제공

필수 요소:

- 브랜드명 `LET'S DAPA`
- 짧은 핵심 카피
- 보조 설명
- 대표 CTA 1~2개
- 배경 그래픽 또는 레트로 질감 비주얼
- 현재 큐레이션 중인 서비스 수 또는 핵심 카테고리 표시

예시 카피 방향:

- `흩어진 업무와 생활 링크를, 한 장의 탐색 화면으로`
- `DAPA를 위한 레트로 무드의 서비스 보드`
- `오늘 필요한 법령, 국회, 청사 정보까지 한 번에`

추천 CTA:

- `서비스 둘러보기`
- `오늘의 바로가기`

## 7.2 카드 섹션 구조

카드 섹션은 단순 버튼이 아니라 `서비스 소개 카드`로 구성한다.

카드 공통 요소:

- 로고 이미지 또는 대표 아이콘
- 서비스명
- 자동 요약 1~2문장
- 해시태그 3~6개
- CTA 버튼
- 외부 링크 여부 표시

카드 탐색 방식:

- 모바일: 세로 스크롤 기반 카드 피드
- 데스크톱: 2열 또는 비대칭 그리드
- 일부 구간은 가로 스크롤 카드 레일도 허용
- 섹션 제목과 태그 필터를 통해 콘텐츠성 탐색 제공

## 7.3 자동 요약 기능

목적:

- 사용자가 링크를 열기 전에 서비스 내용을 즉시 파악하게 함

동작 방식:

- 각 서비스별 사전 정의 요약문을 기본값으로 제공
- 추후 외부 페이지 메타데이터 또는 수동 CMS 방식으로 확장 가능
- 초기 MVP에서는 정적 요약으로 시작해도 됨

요약 가이드:

- 60~100자 내외
- 무엇을 할 수 있는지 중심
- 공공 정보 서비스 특성상 과장 없는 효용 중심

## 7.4 해시태그 기능

목적:

- 서비스의 핵심 단어를 시각적으로 빠르게 전달
- 정보 구조를 더 쉽게 인식하게 함

예시 태그:

- `#방위사업법`
- `#국회실시간`
- `#중고거래`
- `#구내식당`
- `#과천청사`
- `#복지시설`
- `#공공정보`
- `#업무도구`

동작 방식:

- 카드별 고정 태그 노출
- 추후 클릭 시 관련 카드 필터링 기능 확장 가능

## 7.5 로고 이미지 기능

목적:

- 카드별 식별성 강화
- 링크 모음처럼 보이는 느낌을 줄이고 서비스 감도를 높임

구성:

- 각 서비스별 로고 또는 대표 썸네일
- 로고가 없는 경우 텍스트 기반 모노그램 카드 사용
- 레트로 톤에 맞게 단색 또는 절제된 컬러 보정 적용

## 7.6 스크롤 기반 탐색 경험

목표:

- 사용자가 페이지를 넘길수록 새로운 서비스를 발견하는 흐름 제공

요구사항:

- 섹션 전환이 자연스러워야 함
- 카드가 반복되더라도 지루하지 않도록 변주 필요
- 태그, 로고, 요약, CTA가 일정한 규칙 속에서 반복되어야 함

추천 패턴:

- Hero
- Featured Services
- Scrollable Service Cards
- Utility / Daily Life Split Section
- Archive-style Grid
- Home Shortcut Guide
- Footer

## 10. 디자인 방향

기본 조건:

- shadcn/ui 기반의 정돈된 컴포넌트 구조 사용
- 레트로 무드 적용
- 단, 촌스럽거나 과도하게 복고풍이 아닌 `현대적으로 정리된 레트로`

### 10.1 추천 디자인 방향 3안

#### A안. Retro Editorial Archive

특징:

- 크림 화이트 배경
- 블랙 타이포 + 톤다운 레드/브라운 포인트
- 인쇄물 느낌의 여백과 선
- 카드가 잡지 기사처럼 정렬됨

장점:

- TAPTAP 레퍼런스와 가장 자연스럽게 연결됨
- 정보성 서비스도 콘텐츠처럼 보이게 만듦
- 로고, 요약, 해시태그 구조와 궁합이 좋음

단점:

- 지나치게 절제하면 공공기관 문서처럼 딱딱해질 수 있음

#### B안. Retro Dashboard Utility

특징:

- 올드 컴퓨터 터미널/제어판에서 착안한 레이아웃
- 그리드, 라벨, 상태 표시, 모노스페이스 일부 활용
- 올리브, 크림, 차콜 계열 조합

장점:

- 업무 도구 느낌이 강함
- 카드별 상태와 분류를 보여주기 좋음

단점:

- 생활 정보, 거래장터 같은 섹션에서 다소 건조할 수 있음
- 자칫 B2B 관리 화면처럼 보일 수 있음

#### C안. Neo Retro Poster Landing

특징:

- 굵은 타이포와 큰 히어로
- 포스터 같은 강한 비주얼
- 섹션마다 컬러 블록과 강한 대비

장점:

- 첫인상이 강함
- GDWEB 스타일의 시선 집중형 히어로와 잘 맞음

단점:

- 카드 탐색 UI보다 프로모션 랜딩처럼 보일 수 있음
- 장기적으로 서비스 확장 시 피로도가 생길 수 있음

### 10.2 최종 추천 방향

LET'S DAPA에는 `A안 Retro Editorial Archive`를 기본으로 하고, 히어로 섹션에만 `C안 Neo Retro Poster Landing`의 강한 첫인상을 일부 섞는 조합이 가장 적합하다.

추천 이유:

- 서비스 수가 늘어나도 카드 아카이브 구조로 확장하기 쉽다.
- `로고 + 요약 + 해시태그`를 가장 자연스럽게 담을 수 있다.
- 공공/업무 성격을 해치지 않으면서도 서비스처럼 보이게 만든다.
- GDWEB에서 참고할 수 있는 강한 첫 화면 연출을 히어로에만 집중 적용하기 좋다.

## 11. UI/UX 상세 요구사항

- 카드 하나만 봐도 서비스 목적을 알 수 있어야 한다.
- CTA 없이도 요약과 태그만으로 내용을 파악할 수 있어야 한다.
- 스크롤 중 반복 피로가 없도록 카드 크기와 배치를 변주한다.
- 모바일에서 첫 2초 안에 `무엇을 하는 페이지인지` 이해되어야 한다.
- 외부 링크라는 점은 분명히 표시한다.
- 홈 화면 추가 유도는 방해하지 않되 분명하게 보여야 한다.
